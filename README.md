## 📌 Проблема

Государственные субсидии в животноводстве РК распределяются по принципу **«первый подал — первый получил»**: финансирование достаётся тому, кто быстрее собрал документы, пока не кончился бюджет. Реальная эффективность хозяйства и масштаб производства не учитываются.

**Наше решение** переходит к **merit-based скорингу**: система ранжирует заявителей по данным и объясняет каждое решение.

---

## 📊 Данные

Реальный реестр заявок из системы **subsidy.plem.kz** (Министерство сельского хозяйства РК):

| Параметр | Значение |
|----------|---------|
| Период | Февраль — Октябрь 2025 |
| Заявок всего | 36 651 |
| Областей | 18 |
| Районов | 192 |
| Направлений животноводства | 9 (скотоводство, птицеводство, овцеводство...) |
| Видов субсидий | 46 |

---

## 🧠 Подход

### Target-переменная
Используем **исторический статус заявки** как обучающий сигнал:
- `1` = субсидия выдана (Исполнена / Одобрена / Сформировано поручение) — 91%
- `0` = отказ (Отклонена) — 9%

### Признаки
| Признак | Описание |
|---------|---------|
| `log(Причитающая сумма)` | Масштаб заявки |
| `log(Норматив)` | Ставка субсидии на голову |
| `log(поголовье)` | Расчётное кол-во голов = сумма/норматив |
| `Область` | Регион хозяйства |
| `Направление водства` | Вид животноводства |
| `Наименование субсидирования` | Тип субсидии (46 видов) |
| `Район хозяйства` | Район расположения |

### Модель: Gradient Boosting Classifier
- 300 деревьев, learning_rate=0.05, max_depth=4
- Train/Test: 80/20, стратифицированный split
- **ROC-AUC = 0.80**

## 🔍 Explainability

Каждая заявка получает:
- **Score** от 0 до 1
- **Рекомендацию**: `✅ Рекомендовать` / `⚠️ На рассмотрение` / `❌ Отклонить`
- **Ранг** внутри области
- **Топ-3 фактора**, повлиявших на решение

> ⚠️ Финальное решение остаётся за комиссией. Система — инструмент поддержки, не замена эксперту.

## 🏗️ Архитектура

```
Реестр заявок (subsidy.plem.kz)
    → Preprocessing (фильтрация, очистка)
    → Feature Engineering (log-трансформации, encoding)
    → Gradient Boosting (обучение на историческом target)
    → Scoring Output (score + рекомендация + объяснение)
    → Комиссия (финальное решение)
```
